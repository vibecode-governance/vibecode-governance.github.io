Maturity Model & Roadmap

Self-assessment and 30/60/90 day implementation plan

Governance isn't a project — it's a practice. You don't "finish" and move on. You start where you are, activate the highest-value controls first, and build capability iteratively. This page gives you a self-assessment to find your starting point and a phased plan to get to operational maturity.

The 80/20 of AI governance: Activating Shield + Trust Layer + Code Analyzer in your CI pipeline addresses roughly 80% of the technical risk surface. The remaining 20% is organizational — people practices, design disciplines, and review rituals — and takes longer to build but compounds over time.

Self-Assessment: Where Are You Today?

Answer these questions honestly. Your answers determine your starting point in the 30/60/90 plan. Most organizations starting AI-assisted development will be at Level 1 or 2 for most pillars — that's expected. The point is to know where you are so you can prioritize.

The 30/60/90 Day Plan

DAYS 1–30

Foundation: Activate Platform Controls

Focus on controls that are platform-native and can be activated without organizational change. This phase is primarily configuration — Shield products, Trust Layer, and pipeline automation.

Shield Activation

Enable Platform Encryption on revenue fields, competitive data, and customer PII. Enable Field Audit Trail on Opportunity, Account, and Case objects. Activate Event Monitoring and configure initial log retention.

Trust Layer Configuration

Confirm Trust Layer is active for all Agentforce agents. Verify PII masking rules. Review injection defense settings. Enable audit logging for all LLM interactions.

Pipeline Security

Add Code Analyzer to CI/CD pipeline in blocking mode. Create minimum rule set: with sharing required, no naked SOQL/DML, USER_MODE enforcement. Establish baseline violation rate.

Data Classification

Run Data Detect to identify sensitive data locations. Document fields that agents currently access. Create initial field sensitivity inventory. Identify gaps between agent access and data sensitivity.

Exit Criteria for Phase 1

Shield products active and logging. Trust Layer audit trail generating data. Code Analyzer blocking deploys on security violations. Data classification inventory complete. You now have visibility — you can see what's happening.

DAYS 31–60

Process: Establish Governance Practices

With platform controls active, focus shifts to the practices that use them effectively. This phase introduces human processes: design standards, review rituals, agent governance policies, and initial metrics.

Design Discipline

Establish spec-before-code requirement for all AI-assisted work. Create initial prompt template library (5-10 templates for common patterns). Define acceptance criteria standards. Begin tracking spec completion rate.

Review Standard

Implement the 5-question AI Code Review Standard. Train reviewers on AI-specific failure patterns. Establish review SLA (target: <8hr PR cycle time). Begin tracking rejection patterns.

Agent Governance

Implement bounded autonomy: confirmationRequired on financial and destructive actions. Define action catalogs for each agent. Configure escalation triggers. Test emergency kill switches.

Metrics Baseline

Begin measuring TrueThroughput (reworkless output per sprint). Establish Code Turnover baseline (<12% target for 30-day window). Track Maintainability Index on AI-generated code. Set initial targets.

Exit Criteria for Phase 2

Spec-before-code practiced on all new work. AI Code Review Standard operational. Agent bounded autonomy configured. TrueThroughput being tracked. You now have control — practices prevent problems before they reach production.

DAYS 61–90

Culture: Embed Governance in the Organization

Platform controls are active, processes are operational. Now embed governance into career incentives, hiring, and organizational culture. This is the phase that makes governance sustainable — it becomes how the team works, not a checklist imposed on them.

Career Path

Publish the updated career ladder (Curator → Pipeline Owner → Constraint Architect). Align performance reviews with governance contributions. Recognize "taste" demonstrations. Make constraint authorship a promotion signal.

Hiring & Interview

Implement the three-stage AI-era interview (constraint definition, debugging hallucinations, demonstrating taste). Update job descriptions to reflect new competency model. Remove "years of experience writing code" as primary signal.

Continuous Improvement

Review first 90 days of Event Monitoring data. Identify recurring agent behavioral patterns. Refine Transaction Security Policies based on actual data. Update prompt templates from rejection patterns.

Operational Health

Establish governance review cadence (monthly). Assign pillar owners. Create escalation playbooks. Conduct first tabletop exercise (simulated agent incident). Document lessons learned and feed back into practices.

Exit Criteria for Phase 3

Career incentives aligned with governance. Hiring process updated. First 90 days of operational data reviewed and policies refined. Governance review cadence established. The organization is now self-improving — governance gets better over time through its own feedback loops.

Quick-Start: If You Can Only Do Three Things

If your team is starting AI-assisted development next week and you need immediate governance, activate these three controls today:

1. Code Analyzer in blocking mode. AI-generated Apex cannot reach production without passing security analysis. This single control prevents the most common and most dangerous failure mode (security bypass).

2. Trust Layer with audit logging. Every agent-to-LLM interaction is screened and logged. You get injection defense, PII masking, and a complete forensic trail from day one.

3. confirmationRequired: true on financial actions. Any agent action that moves money, changes pricing, or modifies revenue data requires human confirmation. Limits blast radius while you build fuller governance.

The compounding principle: Each phase builds on the previous one. Platform controls (Phase 1) give you visibility. Processes (Phase 2) give you prevention. Culture (Phase 3) gives you sustainability. Skip a phase and the later ones won't hold — you can't enforce practices you can't monitor, and you can't sustain practices that aren't incentivized.

Summary

The velocity is a gift. The governance gap is the bill.

AI-assisted development is shipping code 10x faster. But 40-62% of that code contains security flaws, and 56% of developers admit they aren't reviewing it line by line. Speed without governance is just faster damage.

The platform is not just where the risk lives. It's where the controls live.

Salesforce already has governance primitives purpose-built for this moment — injection defense, bounded autonomy, real-time blocking, data isolation, deterministic agent logic. The question isn't whether the controls exist. It's whether you've activated them.

In the agentic era, your share of the shared responsibility model just got bigger, not smaller.

Salesforce secures the platform. You secure what you build on it. And you're now building more, faster, with less review than at any point in history. That's not a reason to panic — it's a reason to activate the governance layer that's already there.
