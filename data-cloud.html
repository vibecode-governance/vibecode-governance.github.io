Data Cloud

Trusted data foundation for AI grounding and identity resolution

Note: Data Cloud is being renamed to Data360 in upcoming releases.

Every AI agent is only as trustworthy as the data it reasons over. Data Cloud provides the governed data foundation that grounds AI in accurate, classified, access-controlled information — ensuring agents and AI-generated code work from a single source of truth rather than stale, fragmented, or unauthorized data.

Data Cloud's role in the governance framework is foundational. It sits beneath every other cloud as the layer that unifies customer data, enforces classification and access policies, provides grounding context for LLM interactions, and maintains the lineage trail that makes AI decisions auditable. Without a governed data layer, every other control is working on uncertain ground.

AI governance is ultimately data governance. You can constrain the agent, validate the code, and bound the autonomy — but if the data the agent reads is wrong, stale, or improperly classified, the output will be wrong regardless of how well-governed the execution layer is.

Data Cloud as AI Governance Foundation

Data Cloud provides four capabilities that are essential for governing AI-assisted development and agent execution:

Unified Data with Lineage

Agents and AI-generated code work from a single, harmonized view of customer data rather than querying fragmented sources with conflicting values. Every data element carries lineage — where it came from, when it was last updated, and which rules applied.

Identity Resolution with Clear Rules

When AI personalizes or takes action, it needs to know exactly who the customer is. Data Cloud's identity resolution unifies profiles across sources using defined match rules — giving agents and code a trusted, deduplicated identity to work from.

Data Classification & Sensitivity Tagging

Every data element can be classified (PII, PHI, PCI, confidential) and tagged with sensitivity levels. These classifications flow through to access policies — ensuring AI agents only access data appropriate to their role and permission level.

Zero-Copy Architecture

Data can be accessed and grounded from without duplicating it into separate stores. This means access policies are enforced at the source, classification stays consistent, and there's no "shadow copy" that bypasses governance controls.

How Data Cloud Enables Each Governance Pillar

Design-Time: Grounding Context for AI Generation

When developers use AI to generate code that touches customer data, Data Cloud provides the schema, relationships, and classification metadata that should inform the generation. A governed prompt can reference the actual data model — "this field is classified as PII, apply appropriate handling" — rather than leaving the AI to guess.

Build-Time: Validation Against the Actual Data Model

AI-generated queries and data transforms can be validated against Data Cloud's actual schema before deployment. Does the query reference real fields? Are the join conditions valid? Does the code respect the classification tags on sensitive fields? These checks happen in the pipeline, not in production.

Runtime: Trusted Grounding for Agent Execution

When Agentforce agents need context to make decisions, Data Cloud provides grounded, up-to-date information through a governed access layer. The agent doesn't hallucinate customer details — it retrieves them from the unified profile. And the retrieval respects the same access controls that apply everywhere else on the platform.

Data Controls: Classification and Access at the Source

Data Detect identifies sensitive data across the unified layer. Classification tags propagate through to every downstream consumer. Platform Encryption protects fields at rest. Field Audit Trail tracks every modification. The governance controls live in the data layer itself — not in each application that accesses it.

People: Auditable AI Decisions

When a stakeholder asks "why did the agent recommend this?" or "what data informed this decision?", Data Cloud's lineage capabilities provide the answer. You can trace from an agent action back through the grounding data to the original source — making AI decisions explainable and auditable.

Data Cloud Governance Controls for AI

Consent & Purpose Management

Data Cloud manages consent as a first-class data entity, not just a boolean flag. This means:

Purpose-based access. Data collected for one purpose (e.g., order fulfillment) isn't automatically available for another (e.g., marketing personalization). AI agents inherit these purpose constraints — they can only access data aligned to their defined function.

Consent versioning and audit. When a regulator asks "did this person consent to AI processing at the time of this interaction?", the answer is forensically available. Consent records include timestamps, version history, and the specific scope that was agreed to.

Dynamic enforcement. When consent is revoked, the change propagates through Data Cloud's access policies in real time. Agents that previously had access to that person's data lose it immediately — no manual cleanup required.

Data Quality for AI Grounding

AI agents make decisions based on the data they read. If the data is stale, duplicated, or inconsistent, the decisions will be wrong — regardless of how well the agent logic is governed. Data Cloud provides:

Freshness tracking. Every data element carries a "last updated" timestamp. Agents can be configured to flag or escalate when they're making decisions based on data older than a defined threshold.

Conflict resolution rules. When multiple sources provide different values for the same attribute, Data Cloud's harmonization rules determine which source wins. These rules are explicit and auditable — not hidden in code.

Completeness monitoring. For AI grounding to be reliable, the unified profile needs to be sufficiently complete. Data Cloud can surface where profiles are thin (missing key attributes) so that agent recommendations are appropriately caveated.

Segmentation & Activation Governance

Data Cloud powers audience segmentation for Marketing Cloud, agent grounding for Agentforce, and analytics across all clouds. Governance controls on segmentation and activation include:

Segment definition audit trail. Every segment query is versioned. When an AI-generated segment produces unexpected results, you can trace exactly what criteria were applied and when they changed.

Activation policies. Not all data is appropriate for all destinations. Activation policies control which data can flow to which channels — preventing sensitive fields from being activated to advertising platforms or external systems without explicit approval.

Data minimization. Agents and activations receive only the fields they need for their function. The full unified profile isn't exposed to every consumer — each gets a scoped view based on its purpose and permission level.

The data governance principle: In the AI era, the data layer isn't just storage — it's the trust boundary. Data Cloud ensures that every AI interaction, every agent decision, and every AI-generated query operates on data that is unified, classified, consented, and access-controlled. This is what makes the platform a governance layer rather than just an execution environment.
